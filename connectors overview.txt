## Future-Proof Connector Architecture

### Recommended Technology Stack:

**Core Components:**
```
1. Connector Interface Layer (Python abstract base class)
2. Protocol Handlers (REST, SPARQL, OAI-PMH, IIIF, ArcGIS)
3. Authentication Manager (API keys, OAuth, Basic Auth)
4. Rate Limiter (per-source configurations)
5. Cache Layer (Redis/database)
6. Data Normalizer (converts to unified schema)
7. Error Handler & Retry Logic
```

**Protocol Support Needed:**
- **REST APIs** (90% of sources) - `requests` or `httpx` library
- **SPARQL endpoints** (RDF triple stores) - `SPARQLWrapper` library
- **OAI-PMH** (metadata harvesting) - `Sickle` library
- **ArcGIS REST** (ESRI services) - `arcgis` Python API
- **GraphQL** (some modern APIs) - `gql` library
- **IIIF** (images) - `iiif` library
- **MediaWiki** (Wikimedia) - `pywikibot` or `mwclient`

**File Download Support:**
- Direct HTTP downloads
- Bulk CSV/JSON ingestion
- XML parsing (TEI, EpiDoc, METS)
- GeoJSON/Shapefile processing
- 3D model formats (OBJ, GLB, USDZ)
